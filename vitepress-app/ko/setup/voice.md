# Clova Speech (회의록 작성)

Naver Clova Speech API를 연동하면 KIRA가 음성을 텍스트로 변환하여 회의록을 작성할 수 있습니다.

::: danger 웹 인터페이스 필수!
Clova Speech 기능을 사용하려면 **웹 인터페이스가 반드시 활성화**되어 있어야 합니다.
웹 브라우저를 통해 음성을 녹음하고 KIRA 서버로 전송하기 때문입니다.

먼저 [웹 인터페이스 설정](/ko/setup/web-interface)을 완료한 후 이 페이지로 돌아와주세요.
:::

## 📋 준비물

- Naver Cloud Platform 계정
- Clova Speech 서비스 신청 및 승인
- ✅ **KIRA 웹 인터페이스 활성화** (필수!)

---

## 🔑 1단계: Clova Speech 신청

### 1. Naver Cloud Platform 가입
1. [Naver Cloud Platform](https://www.ncloud.com/) 접속
2. 회원가입 및 로그인
3. 결제 수단 등록 (필수)

### 2. Clova Speech 서비스 신청
1. 콘솔 > **AI·NAVER API** 메뉴
2. **Clova Speech** 선택
3. **"이용 신청하기"** 클릭
4. 서비스 약관 동의
5. 신청 완료 (승인까지 1-2 영업일 소요)

::: tip 승인 대기
Clova Speech는 승인 후 사용 가능합니다.
승인 완료 시 이메일 또는 SMS로 알림을 받습니다.
:::

### 3. Invoke URL 생성
1. 콘솔 > **AI·NAVER API** > **Clova Speech**
2. **"도메인 등록"** 탭 클릭
3. **"도메인 등록"** 버튼 클릭
   - 도메인: `localhost` (테스트용) 또는 실제 도메인
4. 등록 완료 후 **"Invoke URL"** 복사

예시:
```
https://clovaspeech-gw.ncloud.com/external/v1/13280/eda508206872...
```

### 4. Secret Key 확인
1. 같은 페이지에서 **"인증 정보"** 탭 클릭
2. **Secret Key** 확인 및 복사

::: warning Secret Key 보안
Secret Key는 외부에 노출되지 않도록 주의하세요.
Git에 커밋하지 마세요.
:::

## ⚙️ 2단계: KIRA 설정

### 1. 웹 인터페이스 먼저 활성화

::: danger 순서가 중요합니다!
Clova Speech 설정 **전에** 반드시 웹 인터페이스를 활성화하세요.
:::

1. KIRA 앱 > 환경변수 설정
2. **"웹 서버 / 음성 수신 채널"** 섹션으로 스크롤
3. **WEB_INTERFACE_ENABLED** 토글을 **켜기**로 변경
4. 필요한 웹 인터페이스 설정 완료
5. 설정 저장 및 서버 재시작
6. 웹 서버가 `http://localhost:8000`에서 실행 중인지 확인

자세한 내용은 [웹 인터페이스 설정 가이드](/ko/setup/web-interface)를 참고하세요.

### 2. Clova Speech 섹션 찾기
**MCP 설정** > **Clova Speech**

### 3. 설정 활성화
- 토글 스위치를 **켜기**로 변경

### 4. API 정보 입력
- **CLOVA_INVOKE_URL**: 앞서 복사한 Invoke URL
- **CLOVA_SECRET_KEY**: Secret Key

### 5. 설정 저장
- **"설정 저장"** 버튼 클릭
- 서버 재시작

## ✅ 3단계: 웹 인터페이스에서 사용

### 1. 웹 브라우저 접속
1. 브라우저에서 `http://localhost:8000` 접속
2. Microsoft 365 또는 Slack 계정으로 로그인

### 2. 음성 녹음 시작
1. 웹 페이지에서 **"마이크"** 버튼 클릭
2. 브라우저가 마이크 권한 요청
3. **"허용"** 클릭
4. 녹음 시작 (빨간 점 표시)

### 3. 회의 내용 말하기
회의 중 대화 내용을 그대로 녹음하세요:
```
회의 참석자: 홍길동, 김철수, 이영희
회의 주제: 프로젝트 진행 상황 점검

홍길동: 지난주 작업 내용을 공유하겠습니다...
김철수: 백엔드 API 개발이 완료되었습니다...
이영희: UI 디자인 리뷰가 필요합니다...
```

### 4. 녹음 종료
1. **"정지"** 버튼 클릭
2. KIRA가 자동으로 음성을 텍스트로 변환
3. 회의록 형식으로 정리하여 Slack으로 전송

## 🎯 사용 예시

### 회의록 자동 작성
```
[웹에서 녹음]
"오늘 미팅에서는 신규 프로젝트 일정을 논의했습니다.
개발팀은 다음 주까지 프로토타입을 완성하고,
디자인팀은 UI 시안을 준비하기로 했습니다."

[KIRA가 Slack으로 전송]
📝 회의록

**일시**: 2024-11-20 14:30
**주제**: 신규 프로젝트 일정 논의

**논의 내용**:
- 신규 프로젝트 일정 논의
- 개발팀: 다음 주까지 프로토타입 완성
- 디자인팀: UI 시안 준비

**결정 사항**:
1. 프로토타입 완성 기한: 다음 주
2. UI 시안 준비
```

### 음성 메모
```
[웹에서 녹음]
"내일 고객사 미팅 준비해야 함.
프레젠테이션 자료와 데모 환경 점검 필요."

[KIRA가 메모로 저장 및 알림]
✅ 음성 메모 저장 완료

내일 할 일:
- 고객사 미팅 준비
- 프레젠테이션 자료 준비
- 데모 환경 점검
```

### 인터뷰 녹취
```
[웹에서 녹음]
인터뷰 내용 녹음 → 자동 텍스트 변환 → 정리된 형태로 전달
```

## 🔧 문제해결

### 마이크 권한 오류
- 브라우저 설정에서 마이크 권한 확인
- Chrome: 설정 > 개인정보 및 보안 > 사이트 설정 > 마이크
- `localhost` 또는 해당 사이트의 마이크 권한 허용

### "Web Interface not enabled"
- KIRA 설정에서 WEB_INTERFACE_ENABLED 확인
- 웹 서버가 실행 중인지 확인 (`http://localhost:8000`)
- 로그에서 웹 서버 시작 메시지 확인

### "Invalid Invoke URL"
- CLOVA_INVOKE_URL이 올바른지 확인
- Naver Cloud Console에서 URL 다시 확인
- URL 전체를 복사했는지 확인 (앞뒤 공백 제거)

### "Authentication failed"
- CLOVA_SECRET_KEY가 올바른지 확인
- Naver Cloud Console에서 Secret Key 다시 확인
- Key를 다시 생성하여 입력

### 음성 인식이 정확하지 않음
- 조용한 환경에서 녹음
- 마이크와 입 사이 거리 적정 유지 (20-30cm)
- 명확한 발음으로 말하기
- 배경 소음 최소화

### 인식 언어 설정
- Clova Speech는 한국어에 최적화되어 있습니다
- 영어 인식도 가능하지만 정확도는 한국어가 더 높습니다

## 💡 팁

### 효과적인 회의록 작성
- 회의 시작 시 참석자와 주제 먼저 언급
- 중요한 결정사항은 "결정:", "액션 아이템:" 등 키워드 사용
- 명확하게 발음하여 인식 정확도 향상

### 비용 관리
Clova Speech 요금제:
- 무료: 월 100분
- 유료: 초과 시간당 과금

사용량 확인:
- Naver Cloud Console > 이용 요금 > 상세 내역

### 음질 개선
- 외장 마이크 사용 권장
- USB 마이크 또는 헤드셋 마이크
- 노트북 내장 마이크보다 높은 품질

### 긴 회의 녹음
- 긴 회의는 여러 번 나눠서 녹음
- 각 세션별로 주제 구분
- 30분 단위로 나누는 것 권장
