# 스케줄링

KIRA는 메시지와 작업을 예약할 수 있습니다. 특정 시간에 알림을 받거나, 반복 작업을 자동화할 수 있습니다.

## 기본 사용법

### 일회성 예약

특정 시간에 메시지를 받고 싶을 때:

```
사용자: 내일 오전 9시에 회의 준비하라고 알려줘
KIRA: 내일 오전 9시에 알림을 예약했습니다!
```

```
사용자: 3시간 후에 보고서 마감 리마인드해줘
KIRA: 3시간 후에 리마인드를 예약했습니다.
```

### 특정 작업 예약

알림뿐만 아니라 작업도 예약할 수 있습니다:

```
사용자: 매일 아침 9시에 오늘의 일정을 정리해서 알려줘
KIRA: 매일 오전 9시에 일정 정리 알림을 예약했습니다.
```

```
사용자: 금요일 오후 5시에 이번 주 작업 현황 정리해서 팀 채널에 공유해줘
KIRA: 금요일 오후 5시에 작업 현황 공유를 예약했습니다.
```

## 활용 예시

### 리마인더

```
사용자: 30분 후에 스탠드업 미팅 참석하라고 알려줘
KIRA: 30분 후에 스탠드업 미팅 리마인더를 설정했습니다.
```

### 정기 보고

```
사용자: 매주 월요일 오전 10시에 지난 주 이메일 요약해서 보내줘
KIRA: 매주 월요일 오전 10시에 이메일 요약을 예약했습니다.
```

### 마감 알림

```
사용자: 프로젝트 마감 하루 전에 체크리스트 확인 알림 보내줘
KIRA: 마감 하루 전 체크리스트 알림을 예약했습니다.
```

## 예약 관리

### 예약 목록 확인

```
사용자: 내가 예약한 알림 목록 보여줘
KIRA: 현재 예약된 알림 목록입니다:
      1. 내일 오전 9시 - 회의 준비 리마인드
      2. 금요일 오후 5시 - 주간 보고서 작성
      3. 매주 월요일 - 이메일 요약
```

### 예약 취소

```
사용자: 내일 오전 9시 알림 취소해줘
KIRA: 내일 오전 9시 알림을 취소했습니다.
```

## 팁

### 명확한 시간 지정

- **좋은 예**: "내일 오전 9시", "3시간 후", "금요일 오후 5시"
- **모호한 예**: "나중에", "조만간" (정확한 시간을 지정해주세요)

### 작업 내용 명확히

예약된 작업이 실행될 때 KIRA가 정확히 무엇을 해야 하는지 명확하게 알려주세요:

```
# 좋은 예
"내일 오전 9시에 팀 채널에 오늘 할 일 목록 공유해줘"

# 모호한 예
"내일 아침에 뭔가 해줘"
```

### 반복 작업

정기적인 작업은 반복 예약으로 설정하면 편리합니다:

- 매일 아침 일정 알림
- 매주 금요일 주간 보고
- 매월 1일 월간 리뷰

---

## 제한사항

- 예약된 작업은 KIRA 서버가 실행 중일 때만 동작합니다
- 과거 시간으로는 예약할 수 없습니다
- 복잡한 조건부 예약(예: "비가 오면 알려줘")은 지원하지 않습니다
